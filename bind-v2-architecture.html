Kiến Trúc Phân Lớp Hệ Thống

B.I.N.D V2 vẫn giữ mô hình 3 lớp tách biệt trong một tiến trình duy nhất trên Windows Desktop, nhưng bổ sung thêm Graph Engine Layer và Local LLM Tier để hoàn thiện kiến trúc RAG đồ thị thế hệ mới.

Host Shell · WPF

BIND.App

Window host mỏng C# WPF — WebView2 toàn màn hình, không XAML UI. Quản lý vòng đời, DI (Microsoft.Extensions.Hosting), Serilog, HWND Subclassing cho frameless resize.

UI Layer · SPA

frontend/ React 19 + Vite

SPA offline phục vụ qua virtual host bind.app.local. Graph view dùng D3-force Canvas → nâng cấp Sigma.js v3 WebGL khi node >5,000. Markdown renderer GFM.

Bridge IPC

BIND.Bridge

JSON router hai chiều qua postMessage. 9 handler prefix: chat · graph · agent · mcp · models · config · ingest · entities · vlm. Token Coalescing 30-50ms chống lag 60fps.

Domain Core

BIND.Core

Models, Interfaces (ILlmProvider, IExtractor, IEmbedder, IGraphEngine). Hoàn toàn độc lập framework, không deps ngoài.

Data Store

BIND.Data

SQLite + Dapper + sqlite-vec extension. Vector dimension 1024. 7 Repositories. V2: thêm GraphConnections schema cho LightRAG node/edge storage.

Ingestion Pipeline

BIND.Ingestion

Channel<T> parallel worker pool. V2: thêm VLM OCR path (Gemini Flash) song song với Docling/Pandoc. Florence-2 ONNX, Tesseract 5 OCR.

AI & RAG Engine

BIND.AI

Cloud LLM (ILlmProvider: Gemini/Claude/GPT). BGE-M3 CPU embed. Hybrid Search BM25 + Vector. BGE-Reranker ONNX. V2: LightRAG Graph Engine + Ollama local tier.

IPC & AddIn

BIND.IPC + Revit.AddIn

Named Pipe \\.\pipe\bind-revit. ExternalEvent pattern cho Revit main thread safety. V2: dynamic data stream — sync incremental graph khi model thay đổi.

MCP Revit Agent

THBIM-MCP-Server (Node)

Subprocess Node.js, StdIO MCP. 90+ BIM tools. HTTP 5050 bridge. V2: thêm Graph Query tool — LLM truy vấn đồ thị không gian Revit qua Cypher-like API.

Chiến Lược VLM OCR — Nâng Cấp Ingestion Pipeline

V2 bổ sung VLM OCR Path song song với Docling/Pandoc hiện tại. Thay vì parse text trực tiếp từ PDF, hệ thống render từng trang thành ảnh cao DPI và gửi cho VLM — giải quyết triệt để vấn đề bảng biểu bị vỡ và hình ảnh không có mô tả.

Multimodal VLM OCR — Giải Pháp Tốt Nhất Cho PDF Phức Tạp

Render PDF page → PNG 300 DPI → gửi Gemini 2.5 Flash → nhận Markdown hoàn chỉnh với bảng biểu, caption hình ảnh, và meta-tag trang số.

✅ Ưu điểm
  • Độ chính xác bảng biểu 95-98%
  • Tự động mô tả hình ảnh, biểu đồ kỹ thuật
  • Loại bỏ Header/Footer qua System Prompt
  • Sinh [Page X] meta-tag cho citation
  • Xử lý được tài liệu scan + layout 2-3 cột
⚠️ Nhược điểm
  • Tốn token API (dùng Gemini Flash để tiết kiệm)
  • Cần fallback khi không có internet
  • Chậm hơn Pandoc (~1-3s/trang)
Giải pháp: Dùng Qwen2-VL-7B local (Ollama) làm offline fallback.
C# · VlmOcrExtractor.csBIND.Ingestion
// Render PDF page → image → call Gemini Flash VLM
var pageImage = await PdfRenderer.RenderPageAsync(filePath, pageIndex, dpi: 300);
var base64    = Convert.ToBase64String(pageImage);

var systemPrompt = """
  Bạn là chuyên gia chuyển đổi tài liệu kỹ thuật BIM/AEC.
  Chuyển trang PDF này thành Markdown chuẩn GFM:
  - Bảng biểu → Markdown table (| Header |)
  - Hình ảnh → [Hình: <mô tả chi tiết nội dung kỹ thuật>]
  - Bỏ qua Header/Footer lặp lại
  - Giữ số trang ở định dạng [Page N] cuối trang
  """;

var markdown = await _llmProvider.CompleteWithImageAsync(
    systemPrompt, base64, model: "gemini-2.5-flash");

IBM Docling — Layout-Aware Deep Learning (Giữ Nguyên)

Dùng Table-Transformer + Layout-Analysis models để phân tích cấu trúc trang. Chạy local qua Node.js MCP Tool, không cần internet.

  • Độ chính xác bảng biểu: 85-90%
  • Tốc độ: ~1-2s/trang trên CPU
  • Loại bỏ Header/Footer: dựa trên model Layout Detection
  • Ưu tiên dùng cho: PDF bảng biểu phức tạp có đường kẻ rõ ràng
  • Không mô tả được nội dung hình ảnh (chỉ trích ảnh thô)
Routing logic: Nếu PDF có hình ảnh kỹ thuật → VLM OCR. Nếu PDF thuần bảng biểu + text → Docling. Nếu offline không có API → Docling luôn.

Graph RAG Engine — LightRAG & Neo4j Strategy

Đây là cải tiến kiến trúc cốt lõi của V2 — thay thế custom SimilarityIndexer bằng LightRAG framework battle-tested, giải quyết tận gốc vấn đề node biến mất và chi phí rebuild graph khổng lồ mỗi khi thêm tài liệu mới.

Vấn Đề Hiện Tại (V1)

Tại Sao Node Liên Kết Biến Mất?

V1 tính cạnh đồ thị bằng Cosine similarity giữa centroid embedding của 2 Entity. Khi tài liệu bị trích xuất sai (heuristics vỡ bảng), centroid vector bị lệch → Weight liên kết rơi xuống dưới threshold → node bị cô lập và biến mất trên D3 Canvas.

Lỗi gốc rễ: GraphConnections chỉ là static similarity table — không có incremental update, không có graph traversal thực sự.
Giải Pháp V2 — LightRAG

LightRAG Incremental Graph Engine

LightRAG (MIT License) xây dựng đồ thị tri thức qua LLM entity extraction, lưu node/edge vào SQLite. Hỗ trợ incremental update — chỉ xử lý document mới, không rebuild toàn bộ.

Lợi ích: Tiết kiệm token ×1000 vs Microsoft GraphRAG. Incremental phù hợp với Revit model thay đổi liên tục.

RAG Search Flow — Hybrid + Reranker + Graph

V2 nâng cấp RAG pipeline từ vector-only lên 3-path hybrid search: Vector Semantic + BM25 Lexical + Graph Traversal (mới). Ba path hợp nhất qua RRF, sau đó Reranker sắp xếp lại độ ưu tiên ngữ cảnh.

Hình 3 — V2 Hybrid RAG: 3 đường tìm kiếm song song (Vector + BM25 + Graph Traversal). Nét đứt cyan = Graph path mới trong V2.

Routing Logic — Ollama vs Cloud

Quyết Định Chọn LLM Tier

Sau khi assembly context, hệ thống tự động routing:

  • → Ollama Tier 0: Query đơn giản, context <4K tokens, không có Revit tool-calling
  • → Gemini Flash: AutoTag, tóm tắt, context 4K–32K tokens
  • → Gemini Pro / Claude: Revit Agent, reasoning phức tạp, thinking mode
MCP Tool Retrieval (Không Đổi)

Revit Agent Tool Selection

  • Max 12 tools/lượt gửi LLM
  • Score: Cosine(BGE-M3(query), BGE-M3(name+desc))
  • Lexical boost: +0.20 exact name, +0.10/token >3 chars
  • Denylist: chặn cứng tools nhạy cảm hệ thống
  • Agent loop: max 6 vòng, temp locked 0.2

Quản Trị Rủi Ro Kỹ Thuật

🔴 Rủi Ro Cao

API Key Plaintext trong Config

Tệp bind_config.json chứa API keys dạng thô. Nếu máy nhiễm malware → key bị đánh cắp.

Giải pháp: Windows DPAPI encryption — chỉ đúng Windows account mới giải mã được. Thực hiện trong M4.
🟡 Rủi Ro Trung Bình

Chi Phí VLM OCR Token

Nếu ingest nhiều PDF lớn qua Gemini Flash, token cost có thể tăng vọt (~1000 tokens/trang).

Giải pháp: Chỉ dùng VLM cho PDF có hình ảnh (detect bằng metadata). PDF thuần văn bản/bảng → Docling. Qwen2-VL local làm free fallback.
🟡 Rủi Ro Trung Bình

LightRAG Integration Complexity

LightRAG chủ yếu là Python library. Tích hợp vào .NET cần Python subprocess hoặc tìm .NET port.
