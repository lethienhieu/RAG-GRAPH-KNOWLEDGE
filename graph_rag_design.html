Bản Thiết Kế Hệ Thống Graph RAG Cho BIM

Kiến trúc kết hợp giữa đồ thị tri thức không gian (Spatial Graph) và mô hình ngôn ngữ lớn (LLM) để tối ưu hóa quy trình thiết kế, kiểm tra quy chuẩn và vận hành công trình.

Mục Lục

  • 1. Triết Lý Thiết Kế
  • 2. Lựa Chọn Công Nghệ
  • 3. Sơ Đồ Hoạt Động (Data Flow)
  • 4. Sơ Đồ Kiến Trúc Logic
  • 5. Lập Trình & Cấu Hình
  • 6. Kịch Bản Sử Dụng (Use Cases)

01 // Triết Lý Thiết Kế Hệ Thống

Hệ thống được thiết kế dựa trên một triết lý bất biến: Tối ưu hóa giá trị thương mại bằng cách tận dụng tối đa các giải pháp mã nguồn mở và công cụ miễn phí, loại bỏ hoàn toàn việc phát triển lại các thành phần cốt lõi từ đầu (Don't Reinvent the Wheel).

Đối với ngành xây dựng kỹ thuật số (AEC/BIM), dữ liệu không chỉ có tính chất từ ngữ mà còn mang tính chất tập hợp không gian và liên kết topo. Một tài liệu quy chuẩn hay mô hình 3D Revit không tồn tại độc lập mà liên kết chặt chẽ qua các mối quan hệ hình học. Do đó, hệ thống này thiết lập hai luồng xử lý riêng biệt:

Luồng Dữ Liệu Tĩnh (Static Data)

Xử lý các tiêu chuẩn PCCC, quy chuẩn xây dựng dạng PDF (SCDF, BCA). Sử dụng mô hình đồ thị cộng đồng (Hierarchical Community Graph) để trích xuất tri thức vĩ mô.

Luồng Dữ Liệu Động (Dynamic Data)

Đồng bộ liên tục các thay đổi về cấu kiện, mối quan hệ không gian từ mô hình Revit đang thiết kế (vách, dầm, ống dẫn cơ điện) thông qua cơ chế cập nhật tăng dần (Incremental Graph Updates).

💡 Chiến lược kinh tế: Bằng cách kết hợp Local LLM (chạy offline miễn phí qua Ollama) để xử lý khối lượng tác vụ lớn hàng ngày, và Cloud LLM API (như Gemini Free/Low-cost tier) cho các tác vụ suy luận cực kỳ phức tạp, doanh nghiệp có thể triển khai hệ thống này với chi phí gần như bằng 0 mà vẫn đảm bảo tính bảo mật dữ liệu công trình.

02 // Lựa Chọn Công Nghệ Thương Mại Tự Do

Tất cả các công cụ và thư viện được lựa chọn đều có giấy phép mã nguồn mở (MIT, Apache 2.0, hoặc phiên bản Community được phép sử dụng thương mại miễn phí):

LightRAG MIT

Khung Graph RAG thế hệ mới. Hỗ trợ cập nhật đồ thị tăng dần cực nhanh mà không phải build lại từ đầu. Tiết kiệm token gấp hàng ngàn lần so với Microsoft GraphRAG. Thích hợp cho mô hình Revit thay đổi liên tục.

Neo4j Community GPL v3 / Free

Cơ sở dữ liệu đồ thị số 1 thế giới. Phiên bản Community Edition hoàn toàn miễn phí chạy local/server. Cho phép lưu trữ và thực hiện các câu truy vấn không gian phức tạp (Spatial Topology Queries) cực nhanh.

Ollama MIT

Nền tảng chạy Local LLM offline. Cho phép triển khai các mô hình mã nguồn mở mạnh mẽ như Qwen-2.5-Coder-7B hoặc Llama-3-8B trực tiếp trên máy trạm của kỹ sư, bảo mật 100% dữ liệu thiết kế.

LlamaIndex / LangGraph MIT

Bộ điều phối và kết nối dữ liệu. `PropertyGraphIndex` của LlamaIndex giúp ánh xạ trực tiếp đối tượng Revit C# thành cấu trúc Graph Node lưu trữ vào Neo4j một cách tự động.

B. Mã Nguồn Tích Hợp LightRAG Với Ollama & Neo4j

Đây là cấu hình hoàn chỉnh cho phép trích xuất tài liệu tĩnh và đồng bộ đồ thị Revit động mà không cần xây dựng lại đồ thị từ đầu:

graph_rag_service.py
import os
from lightrag import LightRAG, QueryParam
from lightrag.llm import ollama_model_complete, ollama_embedding
from lightrag.utils import EmbeddingFunc

# 1. Cấu hình thư mục làm việc và cơ sở dữ liệu Graph
WORKING_DIR = "./thbim_graphrag_data"
if not os.path.exists(WORKING_DIR):
    os.makedirs(WORKING_DIR)

# 2. Định nghĩa hàm sử dụng Local LLM (Ollama) miễn phí thương mại
async def local_llm_model_complete(prompt, system_prompt=None, history=[], **kwargs):
    return await ollama_model_complete(
        "qwen2.5-coder:7b", # Mô hình code và logic xuất sắc nhất quy mô 7B
        prompt,
        system_prompt=system_prompt,
        history=history,
        **kwargs
    )

async def local_llm_embedding(texts):
    return await ollama_embedding(
        "nomic-embed-text", # Mô hình embedding miễn phí thương mại chất lượng cao
        texts
    )

# 3. Khởi tạo thực thể LightRAG
rag = LightRAG(
    working_dir=WORKING_DIR,
    llm_model_func=local_llm_model_complete,
    embedding_func=EmbeddingFunc(
        embedding_dim=768,
        max_token_size=8192,
        func=local_llm_embedding
    )
)

# 4. Hàm đồng bộ tăng dần (Incremental Sync) dữ liệu Revit từ MCP Server
def sync_revit_elements(elements_json_list):
    """
    elements_json_list: Danh sách chuỗi text mô tả cấu kiện từ Revit API C# gửi qua.
    Ví dụ: "Wall ID: 124500, Type: Partition, FireRating: 1 Hour, SpatialRelations: Borders Room 302"
    """
    for elem_desc in elements_json_list:
        # Sử dụng cơ chế nạp gia tăng (Không cần build lại đồ thị từ đầu!)
        rag.insert(elem_desc)
    print("✓ Đồng bộ tăng dần thành công!")

# 5. Hàm truy vấn ngữ cảnh Graph kết hợp Vector (Hybrid Query)
def ask_bim_assistant(query_text):
    # Chế độ 'hybrid' kết hợp thông tin cấu trúc Graph và nội dung Vector
    response = rag.query(
        query_text,
        param=QueryParam(mode="hybrid") 
    )
    return response

C. Cấu Trúc Đồ Thị Không Gian (Graph Schema) lưu trên Neo4j

Khi đồng bộ dữ liệu đối tượng từ Revit C# sang Neo4j, mô hình dữ liệu sẽ được thiết lập theo schema liên kết tự nhiên sau:

(:Room {id, name, level})
Đại diện cho các không gian, phòng chức năng trong Revit.
(:Wall {id, type, fire_rating})
Tường bao che hoặc tường phân chia không gian.
[:BORDERS]
Quan hệ không gian: Tường borders (ngăn cách) Căn Phòng.
[:CONNECTS_TO]
Quan hệ kết nối: Ống dẫn MEP connects_to Thiết bị (Chiller/AHU).
[:COMPLIES_WITH]
Mối liên kết luật: Đối tượng BIM tuân thủ Điều khoản SCDF nào.

06 // Các Kịch Bản Thực Tế Đem Lại ROI Lớn

Để tối ưu hiệu quả đầu tư công nghệ, 3 kịch bản thực tiễn tại Singapore sau đây sẽ giải quyết bài toán cốt lõi cho các nhà thầu và đơn vị thiết kế lớn:

1. Độc lập kiểm tra khoảng cách thoát hiểm (Travel Distance Audit)

Kỹ sư chỉ cần hỏi: *"Có phòng ngủ nào trên Tầng 4 có khoảng cách đến cầu thang thoát hiểm vượt quá 20 mét không?"*

Kết quả: AI sẽ duyệt qua sơ đồ kết nối phòng -> hành lang -> cầu thang trên Neo4j, tự cộng dồn chiều dài thực tế, so sánh với SCDF Fire Code PDF và đánh dấu đỏ phòng vi phạm ngay trên màn hình Revit.

2. Kiểm tra xung đột kỹ thuật MEP & Sửa đổi tự động

Khi ống nước đi xuyên qua dầm chính chịu lực, kỹ sư ra lệnh bằng giọng nói: *"Hãy kiểm tra xem dầm này có được phép đục lỗ không. Nếu không, hãy chuyển hướng ống đi vòng xuống dưới."*

Kết quả: Graph RAG tra cứu ghi chú kết cấu dầm chính (chịu lực cực hạn), trả về kết quả "Không được khoan cắt". Thông qua MCP Server, AI gửi lệnh API C# để tự động điều chỉnh tọa độ đường ống đi vòng xuống dưới dầm một khoảng an toàn.

🏆 Kết luận từ Bản thiết kế: Bằng cách kết hợp **LightRAG** (cập nhật tăng dần siêu nhanh), **Neo4j** (quản lý quan hệ không gian tối ưu) và **Ollama** (vận hành miễn phí offline), bạn đã tạo ra một hệ sinh thái BIM thông minh có độ chính xác chuẩn công nghiệp. Giải pháp này hoàn toàn miễn phí thương mại và sẵn sàng tích hợp thẳng vào sản phẩm thực tế của bất kỳ doanh nghiệp AEC nào tại Singapore.
